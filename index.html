Selects data from a table or multiple tables.

DISTINCT filters out duplicate results, while ALL, the default, includes all results.

FROM identifies the table being queried.

GROUP BY groups the the result by the given expression(s).

HAVING filter rows after grouping.

ORDER BY sorts the result by the given column(s) or expression(s).

UNION combines the result of this query with the results of another query.

Example:

SELECT EMPNUM, CITY FROM STAFF UNION SELECT PTYPE, CITY FROM PROJ;

SELECT * FROM ( SELECT C_NAME, C_CUSTKEY, O_ORDERKEY, O_ORDERDATE, O_TOTALPRICE, SUM(L_QUANTITY) FROM CUSTOMER, ORDERS, LINEITEM WHERE O_ORDERKEY IN ( SELECT L_ORDERKEY FROM LINEITEM GROUP BY L_ORDERKEY HAVING SUM(L_QUANTITY) > 300 ) AND C_CUSTKEY = O_CUSTKEY AND O_ORDERKEY = L_ORDERKEY GROUP BY C_NAME, C_CUSTKEY, O_ORDERKEY, O_ORDERDATE, O_TOTALPRICE ORDER BY O_TOTALPRICE DESC, O_ORDERDATE ) WHERE ROWNUM <= 100;

An expression in a SELECT statement, which lets you specify the columns you want to retrieve from the database. All columns in a table may be selected using *. Specify the object from which data is selected.The object can be a table,a SELECT query, multiple tables or a joined table. A table alias can be specified for the table itself. Once the table alias is specified, the original table name cannot be used in other places in the query. A table can join with other tables into one new table.

Example:

TABLE1 T1 LEFT JOIN TABLE2 T2 ON T1.ID = T2.ID

Join Clause

innerJoinClause
outerJoinClause

Joins a table.Both inner join and outer join are supported. The innerJoinClause lets you specify an inner or cross join. The outerJoinClause lets you specify an outer join.

Example:

TABLE1 JOIN TABLE2 ON T1.ID=T2.ID

Inner Join Clause

 
INNER
JOIN tableExpression ON onExpression
CROSS
NATURAL
 
INNER
JOIN tableExpression

Inner joins return only those rows that satisfy the join condition. Specify INNER to explicitly specify an inner join. Use ON keyword to specify a join condition. Also you can use CROSS join an NATURAL join.

Example:

a INNER JOIN on a.ID = b.ID
a CROSS JOIN b
a NATURAL JOIN B

Outer Join Clause

outerJoinType JOIN tableExpression ON expression
NATURAL
 
outerJoinType
JOIN tableExpression

Outer joins return all rows that satisfy the join condition and also returns some or all of those rows from one table for which no rows from the other satisfy the join condition. Specify outerJoinType to specify different types of outer joins. NATURAL keyword together with outerJoinType indicates a NATURAL outer join.

Example:

a NATURAL LEFT JOIN b
a LEFT JOIN b ON a.ID = b.ID

Outer Join Type

RIGHT
LEFT
 
OUTER

The outer join type indicates the kind of outer join being performed. RIGHT indicates a right outer join. LEFT indicates a left outer join. OUTER keyword is optional to specify an outer join.

Example:

a RIGHT OUTER JOIN b

The right hand side of a condition.

singleRowSelect is a SELECT query which only has one row result, that means an aggregate function must be used and only aggregate functions can be used in selectExpression. The aggregate functions are AVG, MAX, SUM, MIN, COUNT.

The conditions "IS [ NOT ]" and "IS [ NOT ] DISTINCT FROM" are null-safe, meaning NULL is considered the same as NULL, and the condition never evaluates to NULL.

When comparing with LIKE, the wildcards characters are "_" (any one character) and "%" (any characters). The database uses an index when comparing with LIKE except if the operand starts with a wildcard. To search for the characters "%" and "_", the characters need to be escaped. The default escape character is " \ " (backslash).

At most one escape character is allowed. Each character that follows the escape character in the pattern needs to match exactly.

Example:

LIKE 'Jo%'
